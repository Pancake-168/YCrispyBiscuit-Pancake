# 前端日志体系设计

> React、Vue 等 TS 项目均可参考此规则。

## 需求

前端的 `console` 日志在浏览器中可以在控制台输出，但打包为桌面端后，这些日志需要保留为真实的日志记录。

## 统一函数体系

日志系统通过 `logger` 提供统一接口：

```ts
logger.info('message')   // 信息
logger.warn('message')   // 警告
logger.error('message')  // 错误
```

前端任意位置需要输出日志时，只需调用上述函数，无需关心运行环境。

## 行为分流

`logger` 内部自动判定当前运行环境：

| 环境 | 行为 |
|---|---|
| Web 浏览器 | 直接调用 `console.log` / `console.warn` / `console.error`，在浏览器控制台输出 |
| Tauri 桌面端 | 将日志写入真实的日志文件，在桌面端 DevTools 中可见 |
